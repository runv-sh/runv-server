Junte-se

Gere uma chave SSH no seu computador e conecte-se a entre@runv.club para fazer seu pedido de entrada na comunidade.

SSH — pedido de entrada entre@runv.club

O que vai acontecer

A conta Unix entre não oferece um shell normal: executa um programa que pede o nome de usuário desejado, e-mail e sua chave pública SSH (uma linha). O pedido fica numa fila no servidor; a equipe revisa e, se for aceito, cria a conta com as ferramentas internas do runv. Não é cadastro automático.

É preciso gerar um par de chaves no seu PC (a privada fica com você; você só envia a pública no fluxo). Abaixo: passos para Linux, macOS e Windows.

Linux (Terminal)

  1. Abra um terminal.
  2. Gere uma chave Ed25519 (recomendado). Substitua o comentário por algo seu (ex.: e-mail ou nome do PC):
ssh-keygen -t ed25519 -C "seu-email-ou-identificador" -f ~/.ssh/id_ed25519_runv

Quando pedir passphrase, você pode definir uma (mais seguro) ou Enter vazio (mais simples — menos seguro se alguém copiar o arquivo).

Mostre a chave pública para copiar no passo final (é uma linha que começa com ssh-ed25519):

cat ~/.ssh/id_ed25519_runv.pub

Para se conectar depois com essa chave (quando você já tiver conta própria), o cliente SSH usa por padrão ~/.ssh/id_ed25519 ou o que você definir em ~/.ssh/config. Para o primeiro contato com entre, basta o comando da seção “Ligar a entre@runv.club”.

macOS (Terminal)

  1. Abra o Terminal (Spotlight: “Terminal”).
  2. Os passos são os mesmos que no Linux:
ssh-keygen -t ed25519 -C "seu-email-ou-identificador" -f ~/.ssh/id_ed25519_runv
cat ~/.ssh/id_ed25519_runv.pub

No macOS recente, o OpenSSH já vem instalado. Se você usar o agente de chaves, pode rodar ssh-add ~/.ssh/id_ed25519_runv após gerar.

Windows (PowerShell com OpenSSH)

No Windows 10 e 11, o cliente OpenSSH costuma estar disponível. Abra o PowerShell ou o Terminal do Windows e verifique: 

ssh -V

Se o comando não for encontrado, em Configurações → Aplicativos → Recursos opcionais instale o Cliente OpenSSH (ou use o guia oficial da Microsoft).

Gere a chave (caminho típico da pasta .ssh no seu perfil):

ssh-keygen -t ed25519 -C "seu-email-ou-identificador" -f $env:USERPROFILE\.ssh\id_ed25519_runv

Mostre a chave pública:

Get-Content $env:USERPROFILE\.ssh\id_ed25519_runv.pub

Obs.: Nunca compartilhe o arquivo sem extensão .pub — esse é o privado. Só a linha do .pub entra no pedido ao entre.

Ligar a entre@runv.club

Com a chave já criada, no mesmo terminal (Linux/macOS) ou PowerShell (Windows):

ssh -i ~/.ssh/id_ed25519_runv entre@runv.club

No Windows PowerShell, equivalente:

ssh -i $env:USERPROFILE\.ssh\id_ed25519_runv entre@runv.club

Se sua chave tiver outro nome ou estiver no caminho padrão (id_ed25519 / id_rsa), você pode omitir -i ou ajustar o caminho. Na primeira conexão, o SSH pergunta se você confia na fingerprint do servidor — confirme se estiver falando com o runv certo.

Siga as instruções na tela até o fim. Se algo falhar, entre em contato com admin@runv.club.

Tipos de chave aceitas

O fluxo do servidor segue a política de provisionamento: em geral Ed25519 (recomendado), chaves ECDSA NIST e, em alguns casos, RSA. Preferência forte por Ed25519.
